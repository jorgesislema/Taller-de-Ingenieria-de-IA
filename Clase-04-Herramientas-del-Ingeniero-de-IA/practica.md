## 1. Antes de empezar: checklist

Marca que tengas todo listo:

- [ ] Tengo Python instalado (`python --version` funciona)
- [ ] Tengo Git instalado (`git --version` funciona)
- [ ] Tengo VSCode instalado y lo puedo abrir
- [ ] Tengo GitHub Desktop instalado (o usare la terminal)
- [ ] Tengo Node.js instalado (`node --version` funciona)
- [ ] Tengo Opencode instalado (`opencode --version` funciona)
- [ ] Tengo una cuenta en GitHub (gratuita, anda a [github.com](https://github.com) si no la tenes)
- [ ] Tengo acceso a al menos un chat de IA (ChatGPT, Claude, Gemini, o el que uses)

## 3. Paso a paso: crear tu repo con IA

### Paso 1: Elegi tu estilo

De los 6 ejemplos de arriba, elegi el que mas se parezca a vos:
1. **Gamer / Streamer** — si te gustan los juegos, Discord, Twitch
2. **Creador de Contenido** — si haces TikTok, YouTube, edits
3. **Artista Digital** — si dibujas, haces fanart, pixel art
4. **Musico / Beatmaker** — si producis musica, beats, sampleas
5. **Estudiante Tech / Maker** — si te gusta la electronica, robotica, Arduino
6. **Profesional en Reinvencion (30+)** — si tenes experiencia laboral y queres sumar IA

### Paso 2: Abri tu chat de IA

Anda a ChatGPT, Claude, Gemini o el que tengas a mano.

### Paso 3: Crea un agente (opcional pero recomendado)

Si tenes cuenta paga en alguna plataforma, crea un agente con estas instrucciones:

```markdown
Eres un asistente especializado en crear paginas de presentacion profesional (portfolios) en GitHub.

Tu tarea:
1. Hacele 5 preguntas al usuario para conocerlo (nombre, profesion/interes, 3 habilidades, 3 proyectos o logros, como quiere que lo contacten)
2. Genera un README.md en formato Markdown listo para copiar y pegar en GitHub
3. El README debe tener: titulo con emoji, seccion "Sobre mi", seccion "Habilidades", seccion "Proyectos", seccion "Contacto", y una frase/filosofia al final
4. Usa emojis con moderacion (1 por titulo/seccion)
5. Mantene un tono profesional pero humano
6. No inventes datos: si el usuario no te dio algo, usa placeholders como [Tu Nombre], [tu-email@ejemplo.com]
```

### Paso 4: Respondele al agente

El agente te va a hacer preguntas. Respondelas con honestidad. No necesitas inventar una super experiencia. Esto es para estudiantes y principiantes. Esta bien decir "estoy aprendiendo", "este es mi primer proyecto", "todavia no tengo experiencia".

Cuanto mas genuino seas, mejor queda el portfolio.

### Paso 5: Copia el resultado

Cuando el agente te entregue el README.md generado, copia TODO el texto (Ctrl+A, Ctrl+C).

### Paso 6: Crea el repositorio en GitHub

1. Anda a [github.com](https://github.com) e inicia sesion
2. Arriba a la derecha, clic en el **+** → **New repository**
3. Completa:
   - **Repository name:** `mi-portfolio` (o el nombre que quieras)
   - **Description:** "Mi pagina de presentacion personal, creada con ayuda de IA"
   - **Public** (para que otros puedan verlo)
   - **NO** marques "Add a README file" (vamos a crear el nuestro)
4. Clic en **Create repository**

GitHub te va a mostrar una pantalla con instrucciones. Busca la seccion que dice:

```
…or create a new repository on the command line
```

### Paso 7: Conecta tu repo local con GitHub

Abrí la terminal (o PowerShell en Windows) y navega a donde queres crear tu proyecto:

```bash
cd Documents
```

Crea la carpeta e inicializa Git:

```bash
mkdir mi-portfolio
cd mi-portfolio
git init
```

Crea el archivo README.md. Podes hacerlo con:

```bash
# En VSCode (recomendado)
code .

# O directamente desde terminal
notepad README.md   # Windows
nano README.md      # Mac/Linux
```

Pega el contenido que te genero la IA, guarda el archivo.

### Paso 8: Hace tu primer commit y push

```bash
# Ver que archivos tenes
git status

# Agregar el README
git add README.md

# Tu primer commit
git commit -m "Mi portfolio inicial, generado con IA"

# Conectar con GitHub (copia la URL de TU repo)
git remote add origin https://github.com/TU-USUARIO/mi-portfolio.git

# Renombrar rama principal a main
git branch -M main

# Subir
git push -u origin main
```

> **Si estas usando GitHub Desktop:** es todavia mas facil. File → Add Local Repository → elegis la carpeta `mi-portfolio`. Escribis el mensaje de commit, clic en Commit, clic en Publish (la primera vez) o Push.

### Paso 9: Verifica

Anda a `https://github.com/TU-USUARIO/mi-portfolio`. Deberias ver tu README.md renderizado como pagina de inicio.

> Si no se ve bien, revisa que el archivo se llame exactamente `README.md` (con mayusculas y minusculas tal cual) y que este en la raiz del repo, no en una subcarpeta.

### Paso 10: Comparti

Copiá la URL de tu repo y compartila en el grupo del taller. Es tu primer proyecto publicado. Celebrate.

## 4. Extra: personalizar tu repo

Si ya tenes el README, aca hay 4 mejoras faciles que podes hacer con IA:

### 4.1 Agregar badges (insignias)

Los badges son esas etiquetas de colores que ves en los README profesionales. Pedile a la IA:

```markdown
Agregale badges de GitHub a mi README.md para mostrar:
- Lenguaje: Markdown
- Estado: Activo
- Licencia: MIT
- Hecho con: ChatGPT
```

La IA te va a dar codigo como este:

```markdown
![Markdown](https://img.shields.io/badge/Markdown-000000?style=flat&logo=markdown)
```

```markdown
Agregame una tabla de contenidos al principio del README con links a cada seccion.
```
